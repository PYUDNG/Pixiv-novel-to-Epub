# 新增功能
## 准备工作
- 首先应当通过记忆了解本项目
- 对于和需求相关但记忆不清楚的部分，可以阅读相关文件结构和代码补充
- 清晰后，进入方案设计阶段

## 方案设计阶段
- 首先给出设计方案，和用户商定后敲定最终设计方案
- 如果是较大的功能（比如一个新的模块），就需要先创建DESIGN.md记录设计方案并创建相应目录结构，DESIGN.md应当放在相应目录结构的根层级下
- 若新增功能较小，则可以跳过这一步；若用户明确要求，依然创建DESIGN.md
- 如果功能很大且实现复杂，则应当在DESIGN.md中包含分阶段实现设计
- 如果上述步骤已完毕（和用户确定，或发现已有DESIGN.md文档），就可以进入首次开发阶段

## 首次开发阶段
- 应当遵循文件命名约定、导入路径规则、代码区组织方式
- 检查是否已创建好目录结构，如果没有先创建
- 按照DESIGN.md进行实现，如果有分阶段实现设计，则一次仅完成一个阶段实现，完成后向用户汇报
- 最终所有文件都完成后，向用户汇报结果
- 如果所有文件均写好了，就可以进入错误修复阶段
  
## 错误修复阶段
- 到这里所有文件都应该已经写完了，此时应当先运行`npx run check`进行vue-tsc类型检查
- 发现类型错误后进行修复，之后再进行检查，循环往复直到没有类型错误
- 如遇问题，随时询问用户，包括但不限于以下场景：
  - 难以修复的错误
  - 架构需要大改
  - 设计问题
  - 等等
- 没有静态类型错误后，就可以进入用户检查阶段

## 用户检查阶段
- 此时应当全部开发完毕且没有类型错误，这时应当向用户汇报，用户会进行实际运行测试，检查有无运行时错误或功能问题
- 按照用户反馈的问题进行修复或修改，必要时可以更新DESIGN.md文档
- 没有问题后，进入结尾阶段

## 结尾阶段
- 全部开发完毕、类型检查通过且用户检查通过后，写入AGENT.md
- 对开发中遇到的对项目的理解和记忆中不一致的部分，更新记忆相关内容；本次开发后新增的内容，也应整合到记忆中
